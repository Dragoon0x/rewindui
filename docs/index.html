One component

Drop it next to your app. It starts recording immediately. Every DOM mutation gets captured with its selector, before/after value, and full undo data.

<Rewind />

// Add it alongside your app.
function App() {
  return (
    <>
      <YourApp />

    </>
  )
}

What gets tracked

Four mutation types cover every change an AI agent can make to your UI.

◨

Style

Inline style changes. Colors, spacing, layout, transforms, opacity.

≡

Attribute

Class changes, data attributes, aria states, src, href.

T

Text

Content edits. Headings, paragraphs, button labels, placeholders.

+−

DOM

Added and removed elements. New sections, deleted components, restructured trees.

Accept and reject

Every mutation gets accept/reject buttons. Reject reverts the change in the live DOM. Accept marks it as approved. You can act on entire commits or individual mutations. Partial accepts are supported: keep the color change, reject the font change.

// Programmatic control
const { acceptCommit, rejectCommit, stats } = useRewind()

// Accept everything the agent did
acceptAll()

// Reject a specific commit and revert its DOM changes
rejectCommit(commitId)

// Export the full changelog
const md = exportMarkdown()

Smart grouping

Mutations within a 500ms window are automatically grouped into a single commit. When an agent updates a component, the style change, class change, and text edit all appear together. Each commit gets an auto-generated label based on what changed and where.

Export the changelog

One click exports the full session as structured markdown or JSON. The output includes every commit, every mutation, selectors, before/after values, and accept/reject status. Paste it into your agent to explain what was approved and what needs to be redone.
